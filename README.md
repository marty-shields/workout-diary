## ğŸ” Bearer Authentication
This API uses JWT (JSON Web Token) bearer authentication to secure endpoints. 

**For Integration Tests:**
Tokens are automatically generated by the test infrastructure during test execution. The `JwtTokenProvider` creates tokens programmatically for testing purposes.

**For Local Development:**
When running the API locally, you'll need to generate tokens using the `dotnet user-jwts` command-line tool. This tool manages app-specific local JWTs without requiring an external authentication server:

```bash
# Generate a new token
dotnet user-jwts create --project src/Api

# List existing tokens
dotnet user-jwts list --project src/Api

# View token details
dotnet user-jwts print {ID} --project src/Api --show-all
```

The generated token will be displayed and can be used in requests by including it in the Authorization header:
```bash
curl -i -H "Authorization: Bearer {token}" https://localhost:{port}/your-endpoint
```

The `dotnet user-jwts` tool securely stores JWT signing keys in your user profile and automatically updates the project's `appsettings.Development.json` with the necessary authentication configuration.
